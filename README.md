# ESTRUTURAS - Padrões não são só código, são fundações

## O Fim da Automação Frágil

A maioria dos projetos de automação de testes não falha por falta de código, mas sim pelo excesso de fragilidade. O cenário é clássico: a interface muda, um seletor se quebra e, de repente, dezenas de testes começam a falhar em efeito cascata. Esse é o famoso "Cemitério da Automação", que assombra empresas de todos os portes.

Vivemos agora a ascensão da Inteligência Artificial nos testes de software. Ela promete mais agilidade, mas traz uma advertência: a IA só é tão eficiente quanto os padrões que segue. Sem uma arquitetura sólida, a IA apenas produz "código espaguete" com mais velocidade.

Este livro apresenta o Oracle-Driven Dialogue Pattern (ODDP), um padrão autoral pensado para o desenvolvimento assistido por IA. O objetivo é desmistificar os padrões de projeto — eles não existem para criar burocracia, mas para servir de base, permitindo que time e tecnologia atuem juntos, sem ruídos.
